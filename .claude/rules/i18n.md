# i18n & Formatting Rules

Always use the locale-aware formatters from `@/lib/i18n` — never use raw `toLocaleString()`, `Intl.NumberFormat`, or `Intl.DateTimeFormat` directly.

## Money

- `formatCurrency({ amountCents, currency?, locale? })` — pass **cents**, it divides by 100 internally.
- Default currency: `USD`. Default locale: from Paraglide `getLocale()`.
- Never divide cents by 100 yourself before calling `formatCurrency`.

## Dates

- `formatDate({ value, locale?, timeZone? })` — short date (e.g., "23 Feb 2026")
- `formatDateTime({ value, locale?, timeZone? })` — date + time
- `formatMonthYear({ value, locale?, timeZone? })` — month + year only
- Default timezone: `UTC`.

## Numbers

- `formatNumber({ value, locale?, options? })` — general number formatting with Intl options.

## Locale resolution

Cookie-first (`APP_LOCALE`) → browser preference → `en-US` fallback. Paraglide's `getLocale()` handles this automatically.
