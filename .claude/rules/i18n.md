# i18n & Formatting Rules

Always use the locale-aware formatters — never use raw `toLocaleString()`, `Intl.NumberFormat`, `Intl.DateTimeFormat`, or `date-fns` directly.

## Money

- `formatCurrency({ amountCents, currency?, locale? })` from `@/lib/i18n/number` — pass **cents**, it divides by 100 internally.
- Default currency: `USD`. Default locale: from Paraglide `getLocale()`.
- Never divide cents by 100 yourself before calling `formatCurrency`.

## Dates

All date functions live in `@/lib/i18n/date`. Never import `date-fns` or `@date-fns/tz` directly — the ESLint `no-restricted-imports` rule enforces this.

- `toISODateString(date)` — local-time `YYYY-MM-DD` string (replaces `.toISOString().split('T')[0]`)
- `todayISODateString()` — today in local time
- `formatDate({ value, locale?, timeZone? })` — short date (e.g., "Jun 15, 2025")
- `formatDateTime({ value, locale?, timeZone? })` — date + time (no seconds)
- `formatDateTimeFull({ value, locale?, timeZone? })` — date + time with seconds
- `formatMonthYear({ value, locale?, timeZone? })` — month + year only
- `formatRelativeTime({ value, now?, locale? })` — relative time (e.g., "3 hours ago")
- `getUserTimeZone()` — returns browser's IANA timezone (e.g., "America/New_York")
- Default timezone: `UTC`.
- Use the `<Timestamp>` component (`@/components/ui/timestamp`) for displaying dates with a tooltip showing UTC, local, relative, and ISO formats.

## Numbers

- `formatNumber({ value, locale?, options? })` from `@/lib/i18n/number` — general number formatting with Intl options.

## Locale resolution

Cookie-first (`APP_LOCALE`) → browser preference → `en-US` fallback. Paraglide's `getLocale()` handles this automatically.
