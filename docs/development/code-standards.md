# Code Standards

Code style, conventions, and structure expectations.

## Formatting and Linting

- Prettier (`pnpm format`): `singleQuote: true`, `trailingComma: 'all'`.
- ESLint (`pnpm lint`): max warnings = 0.
- Markdown lint (`pnpm lint:md`).
- `sort-keys-plus`: Object keys must be alphabetically sorted (all
  files, objects with 3+ keys, case-insensitive).

## Imports

- Absolute imports via `@/`.
- **No barrel files** -- import from concrete module paths.
- `simple-import-sort` enforces ordering.
- No parent relative imports (`import-x/no-relative-parent-imports`).

## Naming and Structure

- Filenames must be **kebab-case** (`unicorn/filename-case`).
- **No redundant prefixes** -- when files live in a descriptive folder,
  don't repeat the folder name. Use `filters/context.tsx`, not
  `filters/filter-context.tsx`.
- Modules live under `src/modules/*`.
- Shared config lives under `src/configs/*`.
- DB schema aggregator: `src/db/schema.ts` (re-exports all module
  schemas and relations).

## TypeScript

- Use `type` aliases over `interface`.
- Prefer inline type imports.
- `@typescript-eslint/no-floating-promises` is a hard error -- every
  Promise must be awaited, `.catch()`-ed, or prefixed with `void`.
- `noUnusedLocals` and `noUnusedParameters` are enabled. Prefix
  unused vars/params with `_` to suppress.

## React

- `react/jsx-sort-props` is enforced.
- `react-hooks/exhaustive-deps` is off; be intentional with deps.
- Use `function` declarations for components, arrow functions
  (`const handler = () => {}`) for event handlers.

## Error Handling

- Use `createError()` from `@/lib/logging/evlog` for structured
  errors. See `docs/development/logging.md`.
- Never use `console`. Use `log` (server) or `clientLog` (client).

## Conventional Commits

Commitlint enforces conventional commit format via a `commit-msg` hook:

```text
type(scope): subject
```

- Allowed scopes: `auth`, `ci`, `config`, `db`, `deps`, `docs`,
  `email`, `env`, `finance`, `i18n`, `infra`, `logging`, `routes`,
  `scripts`, `tests`, `todos`, `tooling`, `ui`.
- Scope is optional (empty allowed), but custom scopes outside this
  list are rejected.
- Header max 200 characters.

## Git Hooks (Lefthook)

`lefthook.yml` runs hooks on commit:

- **`pre-commit`** (parallel): typecheck, lint, lint-md, format.
  Auto-fixes staged files via `stage_fixed: true`.
- **`commit-msg`**: commitlint validates the message format.

## Generated Files

Do not edit these files manually:

- `src/modules/auth/db/schema.ts` -- Generated by Better Auth CLI,
  then postprocessed (`pnpm schema:auth`). The postprocess step
  normalizes timestamps, removes explicit column names, strips
  generated relations, and runs ESLint + Prettier.
- `src/routeTree.gen.ts` -- Auto-generated by TanStack Router.

## Module Layout

Recommended per-feature layout (create folders as needed):

```text
src/modules/<feature>/
  api/
  components/
  db/
  emails/
  hooks/
  lib/
  types.ts
```

- Keep feature UI inside the module; route files should compose it.
- Server functions live in `api/`. Import from there rather than
  defining inline in route files.
